# LLM 日常效率助理能力实测 Prompt（改进版 v2）

## 设计说明（给出题者看，不给被测模型看）

### 各 Round 考察重点

- **Round 0**：指令遵从 + 格式能力（冷启动，不给任何任务，看模型怎么处理空日程）
- **Round 1**：信息提取 + 时间推理（高密度口语输入，夹杂无关废话和隐含约束）
- **Round 2**：冲突检测 + 抗干扰（故意制造一个隐藏冲突，不告诉模型怎么解决，看它能不能自己发现并追问）
- **Round 3**（可选加难）：多重矛盾 + 优先级推理（同时给多个冲突的修改，看模型的极限）

---

## 正式 Prompt 开始

---

### Round 0（冷启动）

```
你现在的身份是我的私人效率助理。帮我建一个本周（周一至周日）的日程管理文件，保存到当前目录 schedule.md。

【核心规范——后续所有轮次必须严格遵守】

1. 零废话：不输出任何解释性文字、寒暄或确认语句，仅通过工具操作文件。如果需要向我提问（比如发现冲突），则只输出提问本身，不要附带多余解释。

2. 表格格式：Markdown 表格，四列——
   - 状态：[ ] 或 [x]
   - 日期：具体星期几 + 日期
   - 时间：精确到分钟的起止时间（如 14:45-16:15）
   - 任务内容：简要描述

3. 更新日志：表格下方保留 `### 更新日志` 区块。每次文件被修改，必须自动追加一条记录，格式为 `- [修改时间] 简述变更内容`。首次创建也要记录。

4. 冲突处理铁律：如果新指令会导致任何时间段重叠（哪怕只重叠1分钟），绝对禁止自行决定删除、移动或覆盖任何已有任务。必须停止操作，向我列出具体冲突，等我明确指示后再改。

现在先创建一个空的日程模板（本周七天，暂无任务），让我看看格式对不对。
```

---

### Round 1（高密度信息提取 + 时间推理）

> **设计意图**：一段话里塞了 6 个任务，口语化表达、信息乱序、部分时间模糊需要模型自己推算、夹杂废话和情绪表达。好的模型应该一个不漏地全部提取出来，并且自行推算出合理的起止时间。

```
行，模板没问题。接下来我把这周大概要干的事儿跟你说一下，你帮我排进去：

首先周三上午我得去复查牙齿，挂的号是10:30的，一般看完加上路上来回差不多得两个小时吧；然后B站那期各家大模型横评的视频我拖了好久了必须这周剪出来，素材已经录好了，粗剪精剪加字幕加封面我估计总共得十二个小时左右，你给我拆到周二周三周四的下午去，尽量每天平均分，从下午两点开始搞就行。

对了差点忘了，周五晚上固定跟哥们开黑，雷打不动那种，8:15上线一直到11:30，这个时间谁都不能占。还有个开源项目的PR我得审，大概两个小时的活儿，不太急但周五下班前必须搞完，你自己找个合适的时间塞进去。

周六下午去健身，3点到4:30，周六上午我想去趟宜家买点收纳的东西，大概9:30出门，中午之前能回来。周日彻底躺平别给我排任何东西。
```

> **评分关键点**：
> - 任务共 6 个：牙齿复查、剪视频（拆3天）、开黑、审PR、健身、逛宜家。漏掉任何一个扣分。
> - "10:30 的号 + 来回两小时"→ 应推算出 10:30-12:30。
> - "总共12小时拆到三天下午，从两点开始"→ 每天 4 小时 → 14:00-18:00。
> - 审 PR "两小时 + 周五下班前 + 自己找时间"→ 周五有晚上开黑占了 20:15-23:30，审 PR 应安排在周五白天（如 14:00-16:00 或 9:00-11:00 等）。
> - 宜家 "9:30 出门，中午之前回"→ 9:30-12:00。
> - 周日必须完全空白。

---

### Round 2（隐藏冲突检测——核心鉴别力测试）

> **设计意图**：这一轮故意制造一个模型必须自己发现的时间冲突，而且不告诉模型解决方案。好的模型应该发现冲突后停下来追问；差的模型会直接覆盖或者瞎编一个方案。

```
计划有变：

1. 牙医周三上午临时有事，复查改到周四下午2:45，看完加路程大概还是两小时。
2. 审代码那个对面催了，截止时间从周五下班前提前到周三下午4点前。
```

> **评分关键点**：
> - **必须触发冲突提问**：周四下午原本 14:00-18:00 在剪视频，牙齿复查 14:45-16:45 与其严重重叠 → 模型必须报告此冲突并停止操作，而不是自己决定挪动视频剪辑。
> - 审代码改到周三下午4点前 → 周三下午原本 14:00-18:00 也在剪视频 → 审代码（2小时）想安排在周三下午4点前，意味着最晚 14:00-16:00 开始，但 14:00-18:00 已被视频占据 → 这也是一个冲突，模型如果够细致应该一起报出来。
> - **绝对不能做的事**：未经询问就自行把视频剪辑挪到别的时间。

---

### Round 3（可选加难——多重矛盾压力测试）

> **设计意图**：同时甩出多个修改，其中一条包含自相矛盾的信息，看模型能不能在混乱中保持冷静、逐条处理、该问就问。

```
又有几个变动，一次性跟你说了：

1. 周六健身取消了，改成陪女朋友逛街，时间还是下午3点到4:30。
2. 周五的开黑有个哥们说他那天加班来不了，改到周六晚上8点15开打，但我周五晚上那个时间段还是照常空着别排东西，因为可能临时又改回来。
3. 宜家不去了，周六上午改成在家搞那个开源项目的文档，搞个两三小时差不多。
4. 再加一个：周二上午10点有个线上会议忘了跟你说了，大概一个小时，你补进去。但是周二下午剪视频的时间不要动。
```

> **评分关键点**：
> - 第1条：直接替换即可，无冲突。
> - 第2条：开黑从周五改到周六，但周五原时间段仍保留为空（不安排其他事）。模型需要理解"空着"的含义——不是删掉任务，是保留为预留/待定时段。
> - 第3条：替换宜家为写文档，时间可能需要调整（宜家是 9:30-12:00，写文档"两三小时"可以沿用）。
> - 第4条：周二上午 10:00-11:00 会议，与周二下午剪视频（14:00-18:00）不冲突，正常添加即可。
> - 整体考察模型能否逐条准确处理、不混淆、不遗漏。
