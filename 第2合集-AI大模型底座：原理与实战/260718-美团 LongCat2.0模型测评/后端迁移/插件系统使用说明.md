# DjangoBlog 自研插件系统使用说明

## 一、系统概览

插件系统是一个**基于 Hook（钩子）的轻量级发布/订阅机制**，允许你在**不修改核心代码**的前提下扩展博客功能。

### 核心思想

```
核心代码定义"钩子点(Hook)"  →  插件在钩子上注册回调  →  钩子触发时回调被执行
```

### 文件结构

```
plugins/
├── __init__.py          # 对外导出 API（hooks, apply_filters, do_action, load_plugins ...）
├── apps.py              # Django AppConfig，启动时自动加载插件
├── hooks.py             # HookRegistry：钩子的注册与触发引擎
├── base.py              # BasePlugin：所有插件必须继承的抽象基类
├── loader.py            # 自动发现并加载 plugins/ 下的插件子目录
├── view_count/          # 示例插件 ① 浏览计数
│   ├── __init__.py
│   └── plugin.py
├── seo_optimizer/       # 示例插件 ② SEO 优化
├── article_copyright/   # 示例插件 ③ 版权声明
├── article_recommendation/  # 示例插件 ④ 文章推荐
├── external_links/      # 示例插件 ⑤ 外部链接处理
├── image_lazy_loading/  # 示例插件 ⑥ 图片懒加载
├── reading_time/        # 示例插件 ⑦ 阅读时间估算
└── cloudflare_cache/    # 示例插件 ⑧ Cloudflare 缓存
```

> **约定**：每个插件是一个**独立子目录**，目录内必须有一个 `plugin.py`，`plugin.py` 中必须定义一个名为 `Plugin` 的类，继承自 `BasePlugin`。

## 二、两种 Hook 的区别

系统提供两类钩子，用途不同：

| 类型 | 函数 | 用途 | 典型场景 |
|------|------|------|---------|
| **Filter（过滤器）** | `apply_filters(hook, value, ...)` | 把 `value` 依次传给每个回调，每个回调**必须返回修改后的值**，形成"管道"链 | 加工文章对象（加属性、改正文） |
| **Action（动作）** | `do_action(hook, ...)` | 通知所有回调"某事件发生了"，**不需要返回值**，纯副作用 | 评论发布后发邮件通知、清除缓存 |

> 目前项目已在代码中埋入的 Filter 钩子：`after_article_body_get`（文章对象取出后）。
> Action 钩子（如 `after_comment_post`）已在 `__init__.py` 中预告，尚未在核心代码中触发，**你可以在任意位置调用 `do_action` 来触发**。

## 三、内置钩子点一览

### `after_article_body_get`  ⭐ 已在核心代码触发

**触发位置**：`blog/views.py` → `ArticleDetailView.get_object()`

```python
def get_object(self, queryset=None):
    obj = super().get_object(queryset=queryset)
    obj.viewed()
    # 触发 after_article_body_get 插件钩子
    obj = hooks.apply_filters('after_article_body_get', obj) or obj
    self.object = obj
    return obj
```

**回调签名**：`callback(article, *args, **kwargs) -> article`

**传入的 `article`**：一个 `Article` 模型实例（已从数据库取出）。你可以在它上面**任意附加属性**，模板中都能直接使用。

**执行顺序**：按 `priority` 升序（数字越小越先执行，默认 10）。

| 已注册插件 | priority |
|---|---|
| `view_count` | 10 |
| `seo_optimizer` | 20 |
| `article_copyright` | 30 |
| `article_recommendation` | 40 |
| `external_links` | 50 |
| `image_lazy_loading` | 60 |
| `reading_time` | 70 |
| `cloudflare_cache` | 80 |

## 四、如何启用 / 禁用插件

**只需修改 `DjangoBlog/settings.py` 中的一项**，无需任何代码改动：

```python
# settings.py

# 插件所在的根目录（默认就在项目下的 plugins/）
PLUGINS_DIR = os.path.join(BASE_DIR, 'plugins')

# 启用哪些插件：填插件子目录名称（即 plugin.py 所在文件夹名）
ENABLED_PLUGINS = [
    'view_count',
    'seo_optimizer',
    'article_copyright',
    'article_recommendation',
    'external_links',
    'image_lazy_loading',
    'reading_time',
    'cloudflare_cache',
]
```

- **启用**：把插件目录名加入列表
- **禁用**：从列表中**删除**该字符串即可（下次 Django 重启生效）

> `plugins/apps.py` 的 `PluginsConfig.ready()` 会在 Django 启动时自动调用 `load_plugines()`，读取上面的列表并加载所有启用的插件。

### 运行时动态控制

你也可以在代码中随时加载/获取插件实例：

```python
from plugins import get_plugin, get_all_plugins

# 加载单个插件（立即 import + 注册 + enable）
plugin = get_plugin('view_count')

# 加载全部已启用插件，返回 {名称: 实例} 字典
all_plugins = get_all_plugins()
```

查看当前已注册的所有钩子与回调（便于调试）：

```python
from plugins import hooks
print(hooks.get_registered_hooks())
# {'after_article_body_get': ['Plugin.track_view', ...]}
```

## 五、如何编写一个新插件（逐步教学）

我们要写一个真实的插件：**"文章字数统计"**——在文章对象上附加 `word_count` 属性，模板中显示 "本文共 XXX 字"。

### 第 1 步：创建插件目录

在 `plugins/` 下新建一个文件夹，名字即插件标识：

```
plugins/article_word_count/__init__.py   ← 空文件即可
plugins/article_word_count/plugin.py     ← 必须叫 plugin.py
```

### 第 2 步：编写 `plugin.py`

```python
"""
article_word_count plugin - 统计文章正文字数。
"""

from plugins.base import BasePlugin


class Plugin(BasePlugin):
    # --- 插件元信息（会出现在 get_all_plugins 返回的实例上）---
    name = 'article_word_count'
    description = '文章正文字数统计'
    version = '1.0.0'

    def register(self, registry):
        """插件启用时自动调用的注册方法。

        registry 就是全局单例 hooks（HookRegistry），
        在这里挂载你的回调到想要的钩子上。
        """
        # 挂到 after_article_body_get，优先级 35（在版权之后、推荐之前执行）
        registry.register_hook('after_article_body_get', self.count_words, priority=35)

    @staticmethod
    def count_words(article, *args, **kwargs):
        """实际回调。

        参数:
            article: Article 模型实例
        必须返回:
            article（Filter 钩子强制要求返回加工后的值）
        """
        # 防重复计算：Article 可能被多次取到
        if not hasattr(article, '_word_counted'):
            article._word_counted = True
            # 去掉 HTML/Markdown 标记后统计纯文本字符数
            import re
            plain = re.sub(r'<[^>]+>|[`#*_\[\]()>-]', '', article.body or '')
            article.word_count = len(plain.strip())
        return article
```

> **关键规则**：
> - 类名**必须**是 `Plugin`
> - **必须**继承 `BasePlugin`
> - `register(self, registry)` **必须实现**（挂载回调的地方）
> - Filter 回调**必须返回 value**（这里是 article）
> - 用 `hasattr(article, 'xxx')` 防重复处理是**推荐做法**（多个 view 逻辑可能复用同一对象）

### 第 3 步：在 `settings.py` 启用

```python
ENABLED_PLUGINS = [
    'view_count',
    'article_word_count',   # ← 加这一行
    # ...
]
```

重启 Django，插件立即生效：
- Django 启动 → `PluginsConfig.ready()` → `load_plugins()` → 发现 `article_word_count` → 实例化 `Plugin` → 调用 `register()` → 回调挂上 `after_article_body_get`
- 用户访问文章详情 → `get_object()` → `apply_filters('after_article_body_get', article)` → 你的 `count_words()` 执行 → `article.word_count` 被赋值

### 第 4 步：在模板中使用

打开文章模板（`templates/blog/tags/article_info.html` 或 `article_detail.html`），任意位置插入：

```html
<!-- 显示文章字数（仅插件启用时才有这个属性） -->
{% if article.word_count %}
    <span>本文共 {{ article.word_count }} 字</span>
{% endif %}
```

## 六、高级用法

### 6.1 定义自己的钩子点（在任意核心代码中触发）

假设你要在评论被成功保存后触发一个通知，可以在 `comments/views.py` 的评论写入位置加一行：

```python
from plugins import do_action, apply_filters

# --- 在调用 comment.save() 之前可以用 Filter 加工评论对象 ---
comment = apply_filters('before_comment_save', comment, request=request)

comment.save()

# --- 用 Action 通知所有关心"新评论"的插件 ---
do_action('after_comment_post', comment=comment, request=request)
```

Action 回调签名由你定，注册方式相同：

```python
def register(self, registry):
    registry.register_hook('after_comment_post', self.send_email, priority=10)

@staticmethod
def send_email(comment, request, **kwargs):
    # 发邮件，无需返回值
    if comment.email:
        send_mail(...)
```

### 6.2 在模板标签中使用钩子

由于 `apply_filters` / `do_action` 已在 `plugins/__init__.py` 导出，你也可以在模板触发（如需请在 Django 注册为模板 tag）。推荐在 view 层集中触发，保持模板简洁。

### 6.3 插件生命周期

继承 `BasePlugin` 可覆写三个方法：

```python
class Plugin(BasePlugin):
    def register(self, registry):
        """必须。注册回调。"""

    def enable(self):
        """可选。插件 enable 时调用（默认 pass）。
        适合做一次性初始化：创建缓存键、注册 Django signal 等。"""
        pass

    def disable(self):
        """可选。插件被 disable 时调用（当前 loader 不会主动调，
        但手动调用 instance.disable() 可清理）。"""
        pass
```

### 6.4 隔离性保证

某个插件抛异常**不会拖垮其他插件**——`apply_filters` / `do_action` 内部用 try/except 捕获，错误只会被 logger 记录，不会中断请求：

```python
# hooks.py 内部实现（你无需关心，这里只是说明）
for priority, callback in self._hooks[hook_name]:
    try:
        value = callback(value, *args, **kwargs)
    except Exception as e:
        logger.error('Plugin hook "%s" callback error: %s', hook_name, e)
```

## 七、在模板中消费插件产出的"数据字段"

这是插件系统最实用的场景之一——**在模板中直接读取插件给 `article` 附加的属性**。下面是内置插件在模板中可以使用的字段（前提是插件在 `ENABLED_PLUGINS` 中启用）：

| 字段 | 来源插件 | 含义 | 模板示例 |
|---|---|---|---|
| `article.views` | view_count（或直接字段） | 浏览量 | `{{ article.views }} views` |
| `article.seo_title` | seo_optimizer | SEO 标题 | `<title>{{ article.seo_title }}</title>` |
| `article.seo_keywords` | seo_optimizer | 关键词 | `<meta name="keywords" content="{{ article.seo_keywords }}">` |
| `article.copyright_notice` | article_copyright | 版权声明文本 | `{{ article.copyright_notice }}` |
| `article.related_articles` | article_recommendation | 相关文章 QuerySet | `{% for a in article.related_articles %}` |
| `article.external_links` | external_links | 外链列表 | `{% for link in article.external_links %}` |
| `article.reading_time_minutes` | reading_time | 预估阅读分钟 | `约 {{ article.reading_time_minutes }} 分钟读完` |
| `article.cf_cache_max_age` | cloudflare_cache | CDN 缓存秒数 | （由 view 层读取设置 `Cache-Control` 响应头） |
| `article.body` (改写后) | image_lazy_loading | 给所有 `<img>` 加了 `loading="lazy"` | 自动生效，模板无需改动 |

> 建议给新插件的属性加一个 `_<name>_attached` 哨兵（如内置插件的做法），防止对象在分页、列表等场景被重复注册时重复计算。


### 常见报错排查

| 现象 | 原因 | 解决 |
|---|---|---|
| 插件改了没生效 | Django 未重启 / 不在 `ENABLED_PLUGINS` | 确认列表中加入名称并重启 |
| 插件报"Plugin does not define a Plugin class" | plugin.py 里类名不是 `Plugin` | 类名**严格**叫 `Plugin` |
| "No module named 'plugins.xxx.plugin'" | 目录缺少 `plugin.py` 或目录名与 `ENABLED_PLUGINS` 字符串不一致 | 核对文件名与 `ENABLED_PLUGINS` 名称 |

### 日志

插件加载、插件钩子报错会打到 `djangoblog` logger（`settings.py` 中已配置 `log_file` / `console` 两个 handler），查看 `djangoblog.log` 文件即可。

## 九、快速上手清单 (Checklist)

想让新插件跑起来，**只需 4 步**：

```
1️⃣  在 plugins/ 下建一个子目录，放入 __init__.py + plugin.py
2️⃣  plugin.py 中定义 class Plugin(BasePlugin) 并实现 register()
3️⃣  在 settings.py 的 ENABLED_PLUGINS 列表里加上插件目录名
4️⃣  重启 Django → 完成，插件自动生效
```

完全不用改 `views.py`、`models.py`、`urls.py`。

## 十、项目当前的触发埋点（你若要扩展）

目前 Filter 钩子 `after_article_body_get` **仅**在 `blog/views.py` 的 `ArticleDetailView.get_object()` 中触发一次。如果你想在其他位置也加入钩子，直接在目标位置插入：

```python
from plugins import hooks
result = hooks.apply_filters('after_article_body_get', result) or result
```

即可让所有已注册的插件也在那里生效。设计新的 Hook 名称（如 `before_list_render`）完全自由——只要埋点调用 `apply_filters/hook_name`/`do_action('hook_name')` 即可，HookRegistry 会动态创建。
