# 🚀实战教程3：通过 Tailscale 配置虚拟局域网，并在手机上远程调用本地大模型

本教程将说明如何通过 Tailscale 组网技术，将手机连接到电脑上的 LM Studio，实现跨网络的本地大模型调用。

## 前置准备

- **LM Studio 0.4.6**：已在电脑端安装并下载好测试模型。
- **Tailscale 账户**：用于建立安全的虚拟局域网（免费注册）。
- **手机端 AI 客户端**：如 Chatbox 或 NextChat（应用商店下载）。

## 步骤一：在两端设备部署 Tailscale

建立电脑和手机之间的底层加密连接。

1. **电脑端：** 访问 tailscale.com 下载客户端，使用常规账号（如微软/Google）登录。
2. **手机端：** 在应用商店下载 Tailscale，使用**完全相同的账号**登录。
3. **获取** **IP****：** 在电脑端的 Tailscale 菜单中，找到分配给本机的虚拟 IP（格式通常为 `100.x.x.x`）并记录下来。

## 步骤二：在电脑端配置 LM Studio 服务

将本地模型通过指定的虚拟网卡暴露出来。

1. 打开 LM Studio，加载你准备测试的大模型。
2. 进入左侧 **Local Server** 面板。
3. 在右侧 Server Settings 中进行如下配置：
   1. CORS：开启 (ON)
   2. 在网络中提供服务：开启
   3. Network / Bind Address：修改为 `Any (0.0.0.0)` 或直接选择那个 `100.x.x.x` 的地址。
4. 点击 **Start Server** 启动服务。

## 步骤三：在手机端配置调用环境

在手机上设置 API 接口，对接电脑端的算力。

1. 确保手机上的 Tailscale 处于 **Active** 连接状态。
2. 打开手机端的 Chatbox（或其他客户端），进入设置 -> 模型与提供商。
3. 选择 **LM-Studio/Open AI**。
4. **API** **代理地址：** 填写 `http://100.x.x.x:1234/v1` （替换为电脑的真实虚拟 IP，务必保留 http 和 /v1）。
5. **模型名称：** 手动填入你需要测试的模型名称（例如 `Minimax-M2.5`，这样配置好后，你就可以直接在手机上随时随地测试它在移动端的办公能力或 PDF 解析等实际表现）。
6. **API Key：** 填入任意字符（如 `test`）。

## 步骤四：功能验证与联调测试

关闭电脑的 Wi-Fi 共享，将手机切换到移动数据网络（4G/5G）进行真实外网测试。

- **基础对话测试：** 在手机端发送一段需要复杂链路推理的逻辑问题，观察电脑端的 LM Studio 日志窗口是否有请求接入，以及手机端是否能稳定输出小结。
- **延迟观察：** 在跨网络环境下，首字响应时间可能会略微增加，但持续生成的稳定性应与局域网无异。

## 故障排除指南

| 问题现象       | 可能原因         | 解决方案                                                     |
| -------------- | ---------------- | ------------------------------------------------------------ |
| 手机连接超时   | 电脑防火墙拦截   | 进入 Windows 防火墙，在入站规则中放行 TCP 1234 端口，并确保该规则对“公用网络”生效。 |
| 无法找到主机   | Tailscale 未连通 | 检查手机端 Tailscale 是否处于 Active 状态，且账号与电脑端一致。 |
| 客户端报错 404 | URL 格式错误     | 检查手机端填写的 API 地址，确保包含完整的 http:// 并且以 /v1 结尾。 |
