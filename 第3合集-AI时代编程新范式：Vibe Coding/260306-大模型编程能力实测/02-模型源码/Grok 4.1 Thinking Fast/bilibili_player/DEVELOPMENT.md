# 开发文档

## 项目架构

本项目采用分层架构设计：

```
bilibili_player/
├── src/
│   ├── main.py                 # 应用入口
│   ├── ui/                     # 表示层
│   │   ├── main_window.py      # 主窗口
│   │   ├── video_player.py     # 视频播放器
│   │   ├── comments_widget.py  # 评论组件
│   │   ├── login_dialog.py     # 登录对话框
│   │   └── styles.py           # UI样式定义
│   ├── api/                    # 应用层
│   │   ├── bilibili_client.py  # Bilibili API客户端
│   │   ├── auth_manager.py     # 认证管理
│   │   └── data_models.py      # 数据模型
│   └── utils/                  # 工具层
│       ├── config.py           # 配置管理
│       └── logger.py           # 日志工具
```

## 核心功能实现

### 1. 视频播放
- 使用PyQt6的QMediaPlayer进行视频播放
- 支持进度控制、音量调节
- 集成弹幕显示系统
- 支持全屏播放（预留接口）

### 2. 弹幕系统
- 实时弹幕显示
- 支持发送弹幕（需要登录）
- 弹幕动画效果
- 时间同步显示

### 3. 评论系统
- 分页加载评论
- 支持回复查看
- 发送评论功能（需要登录）
- 实时更新评论列表

### 4. 用户认证
- 支持二维码登录
- 支持账号密码登录
- 自动保存登录状态
- 安全凭据管理

### 5. 收藏管理
- 查看用户收藏夹
- 收藏夹视频列表
- 双击播放收藏视频

## 技术栈

- **UI框架**: PyQt6
- **异步处理**: asyncio + qasync
- **API客户端**: bilibili-api-python
- **数据处理**: Python数据类
- **配置管理**: JSON配置文件
- **日志系统**: Python logging

## 依赖管理

主要依赖包：
- `bilibili-api-python==16.2.0`: Bilibili API封装
- `PyQt6==6.6.1`: GUI框架
- `qasync==0.27.1`: Qt事件循环集成
- `requests==2.31.0`: HTTP请求
- `pillow==10.2.0`: 图像处理

## 配置系统

配置文件位置：`~/.bilibili_player/config.json`

支持的配置项：
- 窗口大小和位置
- 播放器设置（音量、播放模式）
- 登录设置（自动登录、记住密码）
- UI主题设置

## 日志系统

日志文件位置：`~/.bilibili_player/logs/bilibili_player.log`

日志级别：
- DEBUG: 详细调试信息
- INFO: 一般信息
- WARNING: 警告信息
- ERROR: 错误信息

## 扩展性设计

- 异步API设计，支持高并发

## 已知限制

1. **视频播放**: 目前使用系统默认播放器，可能在某些系统上存在兼容性问题
2. **弹幕显示**: 弹幕动画较为简单，可以进一步优化性能和效果
3. **评论加载**: 大量评论时可能影响性能，可以实现虚拟滚动
4. **网络请求**: 没有实现请求重试和缓存机制
5. **国际化**: 目前只支持中文，可以扩展多语言支持
