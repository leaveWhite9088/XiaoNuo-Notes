## 核心结论速览

| 模型/平台 | 白天 @100K TTFT | 夜间 @100K TTFT | 月费 |
|-----------|:--------------:|:--------------:|-----:|
| 硅基流动 · GLM-4.7 | 3.59s | **0.60s** | 按量 |
| GLM-Coding · GLM-5 | 16.50s | 7.85s | ¥149 |
| KIMI-Coding · K2.5 | 17.30s | 11.90s | ¥49 |
| 阿里 · Qwen3.5-Plus | 11.19s | 9.75s | ¥200 |
| MiniMax · M2.7-highspeed | 10.90s | 8.90s | ¥98 |
| 硅基流动 · MiniMax-M2.5 | **8.62s** | 8.00s | 按量 |

**最核心发现：选对平台比选对模型更重要** — 同一 Kimi-K2.5 在不同平台白天 @100K 总耗时从 10.34s 到 104.35s，差距可达 10 倍。

## 场景选型建议

**开发场景（Claude Code / Cursor / Cline）**
- 能力首选：GLM-Coding · GLM-5（对标 Opus 4.6）
- 前端首选：KIMI-Coding · kimi-for-coding（80-90% 截图还原）
- 高吞吐：阿里 · Qwen3.5-Plus（@100K TPS 逆增到 125）

**OpenClaw / 龙虾（消息端智能体）**
- 首选：硅基流动 · GLM-4.7（夜间 TTFT 0.60s 全场最快）
- 月套餐：GLM-Coding · GLM-4.5-air（TTFT 1.11s，TPS 97）

**生产接入 / 系统集成**
- 首选：硅基流动 · MiniMax-M2.5（@100K 8.62s 全场最快）
- 低延迟：硅基流动 · GLM-4.7（TPS 64-77）
- 超高吞吐：GLM-Coding · GLM-4.5-air（TPS 97-121）

## 需规避的组合

- 无问芯穹 · kimi-k2.5（白天 TTFT 38.2s）
- 火山引擎 · glm-4.7 / kimi-k2.5（白天 TTFT 17-29s）
- 阿里 · kimi-k2.5（白天 TTFT 23.3s）
